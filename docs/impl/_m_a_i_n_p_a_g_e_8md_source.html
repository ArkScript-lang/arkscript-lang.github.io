14 ## Enhancing the project
15 
16 ### A brief tour of the architecture of the project
17 
18 * everything under `Builtins` is related to builtin functions, used with the bytecode instructions `BUILTIN id`
19  * adding one will need to reference it in `include/Ark/Builtins/Builtins.hpp` and in `src/Builtins/Builtins.cpp`, + implementing it accordingly under `src/Builtins/[file].cpp`
20 * the Lexer, Parser, AST generation (using Nodes), AST optimizer and Compiler are located under `include/Ark/Compiler/`.
21  * the Compiler calls the Parser on a given piece of code
22  * the Parser calls the Lexer on the given code
23  * the Lexer returns a list of tokens
24  * the Parser returns an AST from the token list
25  * the Compiler calls the AST optimizer on the Parser's AST
26  * the Compiler generated bytecode, which can be used as is by the virtual machine, or be saved to a file
27 * the REPL (read eval print loop) is located under `include/Ark/REPL/`. Basically it's an abstraction level over `replxx` (external library for completion and coloration in the shell) and our virtual machine to run user inputs
28 * the virtual machine lies under `include/Ark/VM/` and all the folders under it
29  * it handles the Closures which capture whole Scopes through `shared_ptr`. Closures are functions with a saved scope, so they can retain information over multiple calls
30  * the Plugin loader is a generic DLL / SO / DYNLIB loader, used by the virtual machine to load ArkScript modules (`.arkm` files)
31  * the Scope is how we store our mapping `variable id => value`, heavily optimized for our needs
32  * the State is:
33  * reading bytecode
34  * decoding it
35  * filling multiple tables with it (symbol table, value table, code pages), which are then used by the virtual machine. It allows us to load a single ArkScript bytecode file and use it in multiple virtual machines.
36  * the State retains tables which are **never altered** by the virtual machines
37  * it can also compile ArkScript code and files on the go, and run them right away
38  * the UserType is how we store C++ types unknown to our virtual machine, to use them in ArkScript code
39  * the Value is a very big proxy class to a `variant` to store our types (our custom String, double, Closure, UserType and more), thus **it must stay small** because it's the primitive type of the virtual machine and the language
40  * it handles constness and type through a tag, alongside the value
41  * it provides proxy functions to the underlying `variant`
42  * the virtual machine handles:
43  * the stack, a single `array<Value, 8192>` (the stack size is a define, thus it can be changed at compile time only)
44  * a pointer to the state, to read the tables and code segments
45  * a pointer to a `void*` user_data, retrievable by modules and C++ user functions
46  * the scopes, and their destruction
47  * the instructions, executed in `safeRun` which is enclosed in a try/catch to display tracebacks when errors occur
48  * external function calls through its private `call` method (to call modules' functions and builtins)
49  * value resolving, useful for modules when a function receives an ArkScript function, through the public method `resolve`

55 ### Using ArkScript
56 
57 An example is often worth a thousands words:
58 
59  #include <Ark/Ark.hpp>
60 
61  int main()
62  {
63  // A state can be shared by multiple virtual machines (note that they will NEVER modify it)
64  // leave constructor empty to select the default standard library (loaded from an environment variable $ARKSCRIPT_PATH/lib)
65  // persistance is needed to use vm.call(function_name, args...)
66  Ark::State state(/* options */ Ark::FeaturePersist);
67 
68  // Will automatically compile the file if needed (if not, will take it from the ark cache)
69  state.doFile("myfile.ark");
70 
71  Ark::VM vm(&state);
72  vm.run();
73 
74  /*
75  If you just want to run a precompiled bytecode file:
76 
77  Ark::State state;
78  state.feed("mybytecode.arkc");
79  Ark::VM vm(&state);
80  vm.run();
81  */
82 
83  /*
84  To run an ArkScript function from C++ code and retrieve the result:
85  we will say the code is (let foo (fun (x y) (+ x y 2)))
86  */
87  auto value = vm.call("foo", 5, 6.0);
88  std::cout << value << "\n"; // displays 13
89 
90  return 0;
91  }

93 ### Adding your own functions
94 
95  #include <Ark/Ark.hpp>
96 
97  Ark::Value my_function(std::vector<Ark::Value>& args, Ark::VM* vm)
98  {
99  // checking argument number
100  if (args.size() != 4)
101  throw std::runtime_error("my_function needs 4 arguments!");
102 
103  auto a = args[0],
104  b = args[1],
105  c = args[2],
106  d = args[3];
107 
108  // checking arguments type
109  if (a.valueType() != Ark::ValueType::Number ||
110  b.valueType() != Ark::ValueType::Number ||
111  c.valueType() != Ark::ValueType::Number ||
112  d.valueType() != Ark::ValueType::Number)
113  throw Ark::TypeError("Type mismatch for my_function: need only numbers");
114 
115  // type is automatically deducted from the argument
116  return Ark::Value(a.number() * b.number() - c.number() / d.number());
117  }
118 
119  int main()
120  {
121  Ark::State state(/* options */ Ark::FeaturePersist);
122  state.doFile("myfile.ark"); // we can call state.doFile() before or after state.loadFunction()
123 
124  state.loadFunction("my_function", my_function);
125 
126  // we can also load C++ lambdas
127  // we could have done this after creating the VM, it would still works
128  // we just need to do that BEFORE we call vm.run()
129  state.loadFunction("foo", [](std::vector<Ark::Value>& args, Ark::VM* vm) {
130  return Ark::Value(static_cast<int>(args.size()));
131  });
132 
133  Ark::VM vm(&state);
134  vm.run();
135 
136  return 0;
137  }

139 ### Adding your own types in ArkScript
140 
141  enum class Breakfast { Eggs, Bacon, Pizza };
142 
143  Breakfast& getBreakfast()
144  {
145  static Breakfast bf = Breakfast::Pizza;
146  return bf;
147  }
148 
149  UserType::ControlFuncs* get_cfs()
150  {
151  static UserType::ControlFuncs cfs;
152 
153  cfs.ostream_func = [](std::ostream& os, const UserType& a) -> std::ostream& {
154  os << "Breakfast::";
155  switch (a.as<Breakfast>())
156  {
157  case Breakfast::Eggs: os << "Eggs"; break;
158  case Breakfast::Bacon: os << "Bacon"; break;
159  case Breakfast::Pizza: os << "Pizza"; break;
160  default: os << "Unknown"; break;
161  }
162  return os;
163  };
164 
165  return &cfs;
166  }
167 
168  int main()
169  {
170  Ark::State state;
171 
172  state.loadFunction("getBreakfast", [](std::vector<Ark::Value>& n, Ark::VM* vm) -> Ark::Value {
173  // we need to send the address of the object, which will be casted
174  // to void* internally
175  Ark::Value v = Ark::Value(Ark::UserType(&getBreakfast()));
176 
177  // register the unique control functions block for this usertype
178  // this cfs block can be shared between multiple usertype to reduce memory usage
179  v.usertypeRef().setControlFuncs(get_cfs());
180 
181  return v;
182  });
183 
184  state.loadFunction("useBreakfast", [](std::vector<Ark::Value>& n, Ark::VM* vm) -> Ark::Value {
185  if (n[0].valueType() == Ark::ValueType::User && n[0].usertype().is<Breakfast>())
186  {
187  std::cout << "UserType detected as an enum class Breakfast" << std::endl;
188  Breakfast& bf = n[0].usertype().as<Breakfast>();
189  std::cout << "Got " << n[0].usertype() << "\n";
190  if (bf == Breakfast::Pizza)
191  std::cout << "Good choice! Have a nice breakfast ;)" << std::endl;
192  }
193 
194  return Ark::Nil;
195  });
196 
197  state.doString("(begin (let a (getBreakfast)) (print a) (useBreakfast a))");
198  Ark::VM vm(&state);
199  vm.run();
200 
201  /*
202  Will print
203 
204  Breakfast::Pizza
205  UserType detected as an enum class Breakfast
206  Got Breakfast::Pizza
207  Good choice! Have a nice breakfast ;)
208  */
209 
210  return 0;
211  }
